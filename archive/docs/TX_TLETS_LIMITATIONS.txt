INSTANCE-SPECIFIC NOTES (confirm on first import)
--------------------------------------------------------------------------------

[ ] #1 -- SexCode / NIBRS reverse-lookup (UNCONFIRMED)
    The codeTypeProvider=NIBRS reverse-lookup (attr ID -> M/F/U) is instance-specific.
    Confirmed working: NJ_NJCJIS, AZ_AZDPS
    Confirmed NOT working: FL_FCIC
    TX_TLETS: NOT YET TESTED
    Action: On first import, test SexCode with attributeTypeId=SEX + codeTypeProvider=NIBRS.
    If CommSys XML shows raw attr ID (not M/F/U): switch to NIBRS_SEX fallback and
    remove sex from RMS QIDM. See KB SEX_CODE_PATTERN.txt.

[ ] #2 -- State / NCIC code translation (UNCONFIRMED)
    Confirm RegistrationState sends 2-letter NCIC code in outbound XML (not attribute ID).
    Pattern: attributeTypeId=STATE + codeTypeProvider=NCIC on QIDM attribute.

[ ] #3 -- dexStateUserId / Badge auto-population (UNCONFIRMED)
    If TX_TLETS requires a badge/ORI field, confirm CommsysGetDexStateUserIdRuleHandler
    auto-populates it from the officer RMS profile. See AZ_AZDPS pattern.
